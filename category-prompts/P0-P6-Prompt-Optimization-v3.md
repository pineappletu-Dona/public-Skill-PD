# P0-P6 品类提示词全面优化方案 v3.0

> **诊断对象**：category-prompts v2.0（19子类目 × 7阶段 = 133段提示词）
> **优化目标**：精炼 + 补强 + 行业对标，打造电商品类分析 SOP 的"终极版本"
> **优化日期**：2026-07-02
> **适用范围**：全行业电商（淘系/抖音/Amazon/跨境通用）

---

## 一、现有框架诊断（5大不足）

### 🔴 不足 #1：P0 缺少决策阻断机制

**现状**：
```
输出：品类机会评分（1-10）+ 关键风险清单（≥3条）+ 推荐切入角度
```

**问题**：
- 只有"评分"没有"go/no-go"阈值
- 没有资金门槛评估（启动资金 2万 vs 15万 差距巨大）
- 没有团队能力匹配度评估（新手 vs 老手适合的品类不同）
- 缺法规准入预检（某些品类需要特殊资质，P0 就应该劝退）

**行业标杆参考**：
- Amazon 选品工具（Helium 10/Jungle Scout）都有"难度评分"维度
- 欧孕 S 级 SOP 有"品类准入红线"机制
- 跨境电商头部卖家采用"三票否决制"（资金/资质/能力任一不通过→换品）

---

### 🟡 不足 #2：P1-P3 数据源太抽象

**现状**：
```
数据源：情报通BI / 生参
数据源：生参搜索排行
```

**问题**：
- 只说工具名，没给**具体操作路径**（去哪里点？导什么字段？）
- 没有数据采集模板（Excel 列名/SQL 查询/API 参数）
- 缺少数据质量校验规则（样本量不足怎么办？数据异常怎么处理？）
- 没有"免费替代方案"（新卖家可能买不起情报通/生参）

**行业标杆参考**：
- 卖家精灵有完整的"数据采集SOP"（截图+步骤）
- 跨境电商有"公开数据反推方法论"（用 BSR 反算销量）
- 头部团队都有内部"数据字典"（每个指标的定义和计算公式）

---

### 🟡 不足 #3：P4 定价模型不够实战

**现状**：
```
定价测算（四级模型）：日常(1.0) / A级(0.9) / A+(0.88) / S级(0.85)
公式：到手价 = 前台价 × 折扣系数 - 优惠券 ...
```

**问题**：
- 公式正确但缺**场景化**（新品期 vs 稳定期 vs 清仓期 定价策略不同）
- 没考虑**竞品动态调价**（竞品降价 10% 怎么跟？）
- 缺利润敏感性分析（成本涨 5%？营销费超 30%？退款率翻倍？还能活吗？）
- 没有定价心理学应用（锚定效应/尾数定价/阶梯定价）

**行业标杆参考**：
- 欧孕 S+ 级 SOP 有"动态定价矩阵"
- Amazon repricing 工具（BQool/Profit Bandit）逻辑
- 淘系"价格带卡位策略"（如何精准卡在竞品下方 1 元）

---

### 🟠 不足 #4：P5 缺风险预案

**现状**：
```
供应商寻源 ≥3家比价
专项验收（各项测试标准）
合同专项（条款要求）
```

**问题**：
- 只有"验收标准"，没有**断供/涨价/品质崩盘的 Plan B**
- 没有供应商备份激活流程（主供出问题，备供多久能顶上？）
- 没有原材料价格波动应对（PP 塑料涨价 20% 怎么办？）
- 缺知识产权风控（OEM 侵权、设计专利撞车）

**行业标杆参考**：
- 苹果供应链管理（多产地备份 + 安全库存缓冲）
- Shein 快反供应链（7天翻单机制）
- 跨境大卖"双供应商制度"（主供 70% + 备供 30%）

---

### 🟠 不足 #5：P6 缺量化 KPI 体系

**现状**：
```
周复盘模板：曝光/CTR/CVR/ACOS/自然单占比/库存周转/Review均分/退货原因分类
```

**问题**：
- 有模板但**没有具体指标阈值**（CTR 多少算好？ACOS 多少该报警？）
- 缺分阶段里程碑（第 1 周/第 4 周/第 12 周分别要达到什么目标？）
- 没有"熔断机制"（什么条件下果断止损清仓？）
- 缺竞品监控频率（多久看一次竞品？看哪些指标？）

**行业标杆参考**：
- Amazon 新品推广"飞轮模型"（曝光→点击→转化→排名→自然流量）
- 淘系"单品运营健康分"（多维度加权评分）
- 抖音电商"GPM（千次成交金额）"核心指标体系

```
  - 团队是否有该品类运营经验？（无经验 = 难度 +50%）

  - 销售窗口是否 < 3个月？（错过等于全年报废）
  - 库存折损率是否 > 25%？

□ 利润空间：目标毛利率是否 ≥ 该品类基准线？
```

#### 【新增 #2】数据质量校验规则

```
## 数据异常检测（⚠️ 发现以下情况需重新核实）

### 异常信号清单
- [ ] 月销售额环比波动 > 50%（可能是统计口径变化或活动爆发）
- [ ] Top 关键词搜索人气突然下降 > 30%（可能是平台算法调整）
- [ ] 价格带出现极端值（如 0.1 元或 99999 元，爬虫错误）
- [ ] Review 数与销量严重不成比例（如月销 1万但 Review 仅 5 条）

### 校验方法
1. **交叉验证**：至少用 2 个数据源对比同一指标
2. **时间序列检查**：画趋势图，肉眼识别异常跳点
3. **合理性判断**：基于行业经验（如母婴类转化率通常 3-8%）

### 数据缺失处理
- 缺失 < 10%：用前后月均值填充
- 缺失 10-30%：标注为"估算值"，谨慎使用
- 缺失 > 30%：该指标暂不纳入决策，改用替代指标
```

```
#### 🟡 中风险事件（发生概率 10-30%，影响程度：严重）

| 风险事件 | 触发条件 | 预案（Plan B） |
|---------|---------|--------------|
| **交期延迟** | 供应商延期 > 7天 | 加急生产（加 10-20%费用）或部分空运补货 |
| ** minor 品质问题** | 次品率 5-15% | 增加入库质检工序 + 供应商扣款 |
| **汇率波动** | 汇率变动 > 3% | 提前锁汇或调整售价对冲 |
| **认证过期** | 证书到期未续 | 提前 90 天启动续期（常规动作） |

#### 🟢 低风险事件（发生概率 < 10%，影响程度：可控）

| 风险事件 | 预案 |
|---------|------|
| 包装破损 | 增加外箱缓冲 + 物流保险 |
| 小批量客诉 | 快速响应（24h 内处理）+ 适度补偿 |

### 备份供应商激活流程（SOP）

```
Step 1: 发现主供异常 → 立即通知采购负责人
Step 2: 查阅《备供名录》（预先筛选并签框架协议的 2-3 家）
Step 3: 向备供发送紧急订单（注明"加急"）
Step 4: 确认交期 + 价格（接受 5-10% 涨价）
Step 5: 同步通知运营团队调整库存计划
Step 6: 主供恢复后评估是否切回（或永久切换）

## 六、总结

### 核心理念升级

**v2.0 的理念**：给你一套提示词模板，你自己去填、去执行
**v3.0 的理念**：给你一套**决策系统**，告诉你：
- ✓ 做什么（明确的 Action Item）
- ✓ 什么时候做（阶段门控机制）
- ✓ 谁来做（RACI 分工）
- ✓ 怎么判断做好了（KPI 阈值 + 熔断机制）
- ✓ 出问题了怎么办（风险预案 Plan B）
- ✓ 这次的经验怎么复用（知识沉淀机制）

### 一句话总结

> **v2.0 是一本"操作手册"，v3.0 是一套"管理体系"。**

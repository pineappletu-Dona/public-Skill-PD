```

③ 合并执行：子类目专属维度 + 通用增强模块 = v3.0 完整提示词
```

```
## 第一部分：🚫 红线否决项（任一触发→建议放弃）

### A. 法规准入
- [ ] 是否需要特殊资质？（参考子类目阈值表的认证要求）
  - 母婴类：A类/GB 31701/CPC/OEKO-TEX
  - 食品类：SC证/ FDA注册
  - 医疗器械：注册证
  - 电器类：3C强制认证
- [ ] 资质获取周期是否 > 3个月？
- [ ] 资质获取费用是否 > 5万元？

### B. 资金门槛
- [ ] 首批备货资金是否 > 可用资金的 60%？
- [ ] 回本周期是否 > 6个月？（参考子类目阈值表）
- [ ] 单品库存积压风险是否 > 30%？

### C. 能力匹配
- [ ] 团队是否有该品类运营经验？（无经验 = 难度 +50%）
- [ ] 供应链资源是否已具备？（从零找供 = 时间 +2月）
- [ ] 售后处理能力是否达标？（高退货率品类需客服团队）

## 第二部分：⚠️ 黄色预警项（需特别关注）

### D. 竞争格局
- [ ] TOP3 品牌市场份额是否 > 60%？（高度垄断）
- [ ] 新品（上架<6月）占比是否 < 10%？（市场固化）
- [ ] 价格战是否已白热化？（毛利率 < 子类目阈值线）

### E. 季节性风险
- [ ] 销售窗口是否 < 3个月？（错过=全年报废）

## 第三部分：🟢 绿色通行信号
```

## ENHANCE-P1: 数据采集实操手册 + 趋势雷达

```
你是电商数据采集专家。请按本模块指导 {品类名称} 的市场数据采集与分析。

本模块解决 v2.0 "数据源太抽象"的问题，提供从免费到付费的三级采集路径。
配合 v2.0 子类目提示词的"大盘概况+关键词分析+专属维度"使用。

## A. 数据源优先级（按成本从低到高）

### Level 1：免费/低成本（推荐起步）

1. **平台公开数据**
   - 淘宝/天猫：搜索结果页销量展示（月销 xxx+）
   - 抖音：商品详情页"已售"数据 + 直播间在线人数
   - Amazon：BSR排名（Jungle Scout免费版/Helium 10 trial）

2. **第三方免费工具**
   - Google Trends（搜索趋势，跨地区对比）
   - 1688 询盘（供应商反馈的市场热度）
   - 社交媒体（小红书笔记数/抖音话题播放量/B站相关视频）

3. **手动采样法**（零成本但耗时）
   - 手动记录 Top 50 产品的价格/销量/Review数
   - Excel统计分析
   - 建议：每周2小时，连续4周建立基线

### Level 2：中成本（月费 200-500元）

1. **卖家精灵**（Amazon专用）
   - 关键词挖掘：搜索量/CPC/竞争度
   - 市场容量：BSR反算销量
   - 竞品监控：价格变动/Listing变更追踪

2. **情报通 BI**（淘系专用）
   - 品类大盘数据
   - 品牌份额
   - 价格带分布

3. **生参**（淘系专用）
   - 搜索词排行
   - 行业热搜榜

### Level 3：高成本（月费1000+元，成熟团队）
1. Datahawk（亚马逊全链路）
2. 欧睿国际（行业研究报告）
3. 尼尔森 IQ（消费者行为）

## C. 数据质量校验规则（⚠️ 发现以下情况需重新核实）

### 异常信号清单
- [ ] 月销售额环比波动 > 50%（统计口径变化或活动爆发）
- [ ] Top关键词搜索人气突然下降 > 30%（平台算法调整）
- [ ] 价格带出现极端值（0.1元或99999元，爬虫错误）
- [ ] Review数与销量严重不成比例（月销1万但Review仅5条）

### 校验方法
1. **交叉验证**：至少用2个数据源对比同一指标
2. **时间序列检查**：画趋势图，肉眼识别异常跳点
3. **合理性判断**：基于行业经验（如母婴类转化率通常3-8%）

### 数据缺失处理
- 缺失 < 10%：用前后月均值填充
- 缺失 10-30%：标注"估算值"，谨慎使用
- 缺失 > 30%：该指标暂不纳入决策，改用替代指标
```

## ENHANCE-P2: 机会量化评分卡 + 痛点映射矩阵 + SWOT

```
你是机会挖掘专家。请按本模块对 {品类名称} 的机会方向进行量化评估。

本模块在 v2.0 "细分空白分析+价格带机会+评论痛点"基础上，新增量化评分与SWOT。
配合 v2.0 子类目提示词的专属维度（如孕产的"孕期阶段空白"、婴童的"月龄段空白"）使用。

## A. 机会方向量化评分（每方向满分100分）

### 评分维度（权重分配）
| 维度 | 权重 | 说明 |
|------|------|------|
| 市场规模 | 25分 | TAM/SAM估算值越大分越高 |
| 增长速度 | 20分 | 近3月CAGR越高分越高 |
| 竞争蓝海度 | 20分 | 竞品数量少/质量差则高分 |
| 利润潜力 | 20分 | 预估毛利率越接近子类目阈值上限分越高 |
| 执行可行性 | 15分 | 团队能做、供应链能配合则高分 |

### 评分模板（至少评估3个机会方向）
| 机会方向 | 市场规模(25) | 增速(20) | 蓝海(20) | 利润(20) | 可行(15) | **总分** | 等级 |
|---------|------------|---------|---------|---------|---------|--------|------|
| 方向1:___ | | | | | | | |
| 方向2:___ | | | | | | | |
| 方向3:___ | | | | | | | |

### 判定规则
- **≥80分**：⭐ 核心机会，必须抓住
- **65-79分**：🟢 优质机会，优先级高
- **50-64分**：🟡 备选机会，资源有余时再做
- **<50分**：❌ 低优先级，暂不考虑

## B. 痛点-解决方案映射矩阵（≥15条）

### 采集要求
- 从 Top10 竞品差评中提炼（每竞品≥50条差评）
- 按提及率降序排列
- 每条必须有可执行的具体改进方案（不能是空话）

### 矩阵模板
| # | 用户痛点(原话提取) | 痛点分级 | 提及率 | 竞品现状 | 改进方案 | 话术草案 |
|---|------------------|---------|-------|---------|---------|---------|
| 1 | | 🔴高频 | 35% | | | |
| 2 | | 🔴高频 | 28% | | | |
| 3 | | 🟡中频 | 18% | | | |
| 4 | | 🟢低频 | 12% | | | |
| ... | (继续到≥15条) | | | | | |

### 痛点分级标准（参考子类目专属维度）
- 🔴 **安全/核心功能类**（必须避免）：母婴-材质异味/甲醛；服饰-色差/缩水；塑料-易碎/异味；大件-运输损坏
- 🟡 **体验类**（改进=卖点）：支撑不足/不透气/易变形/安装难
- 🟢 **场景类**（蓝海机会）：送礼缺包装/尺寸不合/洗涤不便

## C. SWOT 矩阵（针对锁定的1-3个机会方向）

### 优势 Strengths（内部）
- 我们有什么独特资源/能力？
  例：已有该品类供应链资源 / 团队成员有相关经验 / 已有私域用户池

### 劣势 Weaknesses（内部）
- 我们的短板是什么？
  例：无品牌认知度 / 无认证经验 / 资金有限无法大量备货

### 机会 Opportunities（外部）
- 市场给我们的窗口是什么？
  例：竞品普遍反映某痛点 → 差异化机会 / 送礼场景供给不足

### 威胁 Threats（外部）
- 可能击垮我们的风险是什么？
  例：大品牌可能跟进 / 原材料价格上涨 / 政策收紧

### 战略组合
- **SO战略**（优势+机会）：用我们的___快速推出___差异化款
- **WO战略**（劣势+机会）：联合KOL做内容种草弥补品牌劣势
- **ST战略**（优势+威胁）：申请专利保护防止抄袭
- **WT战略**（劣势+威胁）：先小批量测款验证，再逐步加码

## D. Gate 2 准入条件（P2→P3）
- [ ] 机会方向已锁定（≤3个，有量化评分）
- [ ] 痛点-解决方案映射完成（≥15条）
- [ ] SWOT分析完成
- [ ] 团队对方向达成共识

## 输出清单
1. 🎯 机会量化评分卡（≥3个方向，100分制）
2. 😫 痛点-解决方案映射矩阵（≥15条）
```

## 核心理念升级

**v2.0 的理念**：给你一套提示词模板，你自己去填、去执行
**v3.0 的理念**：给你一套**决策系统**，告诉你：
- ✓ 做什么（明确的 Action Item）
- ✓ 什么时候做（阶段门控机制）
- ✓ 谁来做（RACI 分工）
- ✓ 怎么判断做好了（KPI 阈值+熔断机制）
- ✓ 出问题了怎么办（风险预案 Plan B）
- ✓ 这次的经验怎么复用（知识沉淀机制）

### 一句话总结

> **v2.0 是一本"操作手册"，v3.0 是一套"管理体系"。**

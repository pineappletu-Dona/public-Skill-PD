### 1.4 ⭐ 垄断系数（核心指标）

```
=MAX(各品牌总销量) / SUM(市场总销量)
或 =LARGE(品牌销量数组, 1) / SUM(品牌销量数组)

→ 31.5%（品牌A）
```

**判定标准：**

| 垄断系数 | 市场格局 | 新品进入难度 |
|---------:|----------|-------------|
| < 20% | 充分竞争 | ★☆☆ 低 |
| 20%~40% | 有头部品牌，但有机会 | ★★☆ 中 |
| 40%~60% | 头部垄断明显 | ★★★ 高 |
| > 60% | 高度垄断 | ✘ 不建议进入 |

本案例：**31.5% → 中度竞争，有进入机会**

### 1.5 ⭐ 产品定位自动判定

**反推公式：**

```excel
=IF(
    月销量 >= 5000,
    "引流款",
    IF(
        AND(月销量 >= 2000, 价格 <= 60),
        "利基款",
        IF(
            价格 >= 70,
            "利润款",
            "立基款"
        )
    )
)
```

**判定树：**

```
月销量 >= 5000?
├── YES → 引流款（高量低价/中价，拉流量）
│         特征: Price $10-$45, Sale 5000-11000+
│
└── NO ── 价格 >= $70?
            ├── YES → 利润款（高价低量，赚毛利）
            │         特征: Price $70-$117, Sale 850-2300
            │
            └── NO ── 月销量 >= 2000?
                      ├── YES → 利基款（中等体量，细分市场）
                      │         特征: Price $24-$70, Sale 1450-3932
                      │
                      └── NO  → 立基款（低量试水，测试市场）
                                特征: Price $32-$65, Sale 556-866
```

**验证结果：21 个 SKU 中 14 个完全匹配（66.7%）**
- **不匹配的 7 个**属于边界情况（人工微调区域），原始表格可能结合了：
  - BSR排名因素（同销量下 BSR 越小越偏向引流款）
  - Style 因素（油汀/塔式等特殊品类溢价）
  - 主观经验调整

**增强版公式（加入 BSR 权重）：**

```excel
=IF(
    OR(月销量>=5000, AND(月销量>=3000, BSR<=10)),
    "引流款",
    IF(
        OR(AND(月销量>=1500, 价格<=70), AND(月销量>=2000, 价格<=80)),
        "利基款",
        IF(
            OR(价格>=80, AND(价格>=70, BSR<=20)),
            "利润款",
            "立基款"
        )
    )
)
```

## 二、价格区间分析（Sheet4）— 统计公式

### 2.1 K2 汇总文本（复合公式）

**原始输出：** `58美金以下产品共28，市场份额占比11%，平均占比0.39%`

**反推公式（Excel 2019+ / 365）：**

```excel
="$" & 阈值 & "以下产品共"
    & COUNTIF($C$3:$M$42, "<" & 阈值)
    & "个，市场份额占比"
    & TEXT(
        SUMIF($C$3:$M$42, "<" & 阈值, $C$6:$M$42)
        / SUM($C$6:$M$42),
        "0%"
      )
    & "，平均占比"
    & TEXT(
        (SUMIF(...) / SUM(...)) / COUNTIF(...),
        "0.00%"
      )
```

**拆解为 3 步：**

| 步骤 | 公式 | 本例值 | 含义 |
|:----:|------|:------:|------|
| ① 数量 | `=COUNTIF(PriceRange,"<58")` | **28** | 该价位段内产品数 |
| ② 总份额 | `=SUMIF(PriceRange,"<58",SalesCol)/SUM(SalesCol)` | **11%** | 该段占全市场销量比 |
| ③ 平均占比 | `=② / ①` | **0.39%** | 每个产品的平均市占率 |

### 2.2 价格段分布矩阵

对每个价格区间 `[lo, hi)` 计算：

```
数量:   =COUNTIFS(PriceCol, ">="&lo, PriceCol, "<"&hi)
占比:   =COUNTIFS(...) / COUNTA(PriceCol)
销售额: =SUMIFS(SalesCol, PriceCol, ">="&lo, PriceCol, "<"&hi)
销额占比: =销售额 / SUM(SalesCol)
```

本例分布：

| 价格段 | SKU数 | 占比 | 销售集中度 |
|--------|:-----:|:----:|:-----------|
| $0–$20 | 3 | 14.3% | An超低价引流 |
| $20–$30 | 5 | 23.8% | **主战场** — 最大集中区 |
| $30–$50 | 4 | 19.0% | 竞争激烈区 |
| $50–$80 | 7 | 33.3% | **SKU最多区** — 利基+利润混战 |
| $80–$120 | 2 | 9.5% | 高价利润区 |
| $120+ | 0 | 0% | 空白 |

### 2.3 最优价格区间推荐算法

```
最优区间 = ARGMAX( 总销量/竞争强度 )

其中:
  总销量      = SUMIF(PriceCol, 区间条件, SalesCol)
  竞争强度    = COUNTIF(PriceCol, 区间条件)
  区间得分    = 总销量 / 竞争强度   ← 越高越好
```

## 三、上架时间分析（Sheet6）— 老品霸榜率

### 3.1 K2 汇总文本

**原始输出：** `BSR，3年及以上产品共48个，市场份额占比69%，平均占比1.44%`

**反推公式：**

```excel
="BSR，" & 年限 & "年及以上产品共"
    & COUNTIF(AgeCol, ">=" & 年限)
    & "个，市场份额占比"
    & TEXT(
        SUMIF(AgeCol, ">=" & 年限, SalesCol) / TotalSales,
        "0%"
      )
    & "，平均占比"
    & TEXT(②/①, "0.00%")
```

| 步骤 | 公式 | 本例值 |
|:----:|------|:------:|
| ① 老品数量 | `=COUNTIF(AgeCol,">=3")` | **48** |
| ② 老品份额 | `=SUMIF(AgeCol,">=3年",Sales)/TotalSales` | **69%** |
| ③ 平均占比 | `=②/①` | **1.44%** |

**老品霸榜率判定：**

| 老品霸榜率 | 市场特征 | 新品策略 |
|----------:|----------|----------|
| < 40% |新品活跃，轮换快 | 可以直接切入 |
| 40~60% |有老品护城河 | 需要差异化 |
| 60~80% |**老品强势霸榜** | ⚠️ 高风险，需要强差异化 |
| > 80% |几乎被老品垄断 | ✘ 极难进入 |

本例：**69% → 老品强势，需强差异化才能突围**

### 3.2 K24 亚马逊自营占比

**原始输出：** `亚马逊自营共11个产品，占比3.57%`

```
自营数量: =COUNTIF(SellerTypeCol, "Amazon")       → 11
自营占比: =SUMIF(SellerTypeCol, "Amazon", Sales)/Total  → 3.57%
```

**自营占比判定：**

| 自营占比 | 含义 |
|--------:|------|
| < 5% | Amazon 未关注此品类，安全 ✅ |
| 5~15% | Amazon 有布局但不主导 |
| > 15% | Amazon 重仓此品类，危险 🚨 |

本例：**3.57% → 安全，Amazon 未垄断该类目**

### 4.4 ⭐ BSR → 月销估算公式（幂律模型）

从品牌 A 的实际数据反推：

| BSR | 月销 | k 值 (log(11373/sale)/log(bsr)) |
|----:|-----:|:---:|
| 1 | 11,373 | — (基准) |
| 7 | 5,372 | **0.385** |
| 11 | 2,697 | **0.600** |
| 14 | 2,302 | **0.605** |
| 40 | 650 | **0.776** |

**平均 k ≈ 0.59**（品类相关，不同品类需重新拟合）

```
通用估算公式：
  月销 ≈ BSR#1的月销 / (BSR排名 ^ 0.59)

或更通用的参数化版本：
  月销 = a × (BSR ^ (-k))

  其中:
    a = 第1名月销（基准销量）
    k = 幂律指数（0.5~0.8，因品类而异）
      - 标准品/日用品 → k 较高 (~0.7)，头部效应强
      - 非标品/长尾品 → k 较低 (~0.5), 分布较均匀
```

## 五、利润分析（推断）

### 5.1 毛利计算公式

```
售价 = FBA价格 (含Amazon佣金)
成本 = FOB采购费 + 头程运费 + FBA费用 + 退货损耗率
毛利 = 售价 - 成本
毛利率 = 毛利 / 售价
ROI = 毛利 / 投入成本
```

### 5.2 多维度利润对比表结构

| 维度 | 公式 |
|------|------|
| 单件毛利 | `=售价 - (FOB + 头程 + FBA + 仓储 + 退货*售价*退货率)` |
| 毛利率 | `=单件毛利 / 售价` |
| 月净利润 | `=单件毛利 * 月销量 - PPC广告费 - 促销折扣` |
| 投资回报周期 | `=总投入 / 月净利润`（月） |
| 盈亏平衡点 | `=固定成本 / (售价 - 变动成本)` |

## 七、使用说明

### 在新项目中使用这些公式

1. **数据源准备**：从卖家精灵/JS 导出 Top100 listing 数据
2. **填入模板**：将 Price / BSR / Sales / Brand / Age / SellerType 填入对应列
3. **自动计算**：以上公式会自动输出：
   - 垄断系数 → 判断能不能进
   - 定位矩阵 → 决定做什么款式
   - 价格分布 → 找最优定价区间
   - 老品霸榜率 → 判断新老品竞争格局
   - 自营占比 → 判断 Amazon 威胁度

### 注意事项

- **定位判定公式匹配率 67%**，边界情况需人工复核
- **幂律指数 k 因品类而异**，首次使用新品类时需用前 5 名数据校准
- **容量标准是经验值**，不同季节可能浮动 ±20%
- 所有百分比均基于**当前样本**，样本越大越准确
